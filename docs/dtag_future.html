doodle tag.dtag notes

This page summarizes how DoodleTag shares .dtag macro files today, how iOS intake currently works, and what a future public-sharing path could look like.

Current Implementation

DoodleTag currently shares macros as raw .dtag files.

  • a user records or exports a macro
  • DoodleTag shares the raw .dtag file
  • another device with DoodleTag can open that file

This path works best when both devices already have DoodleTag installed and the host app hands the file through correctly.

Current iOS Behavior

Files and Messages behave differently on iOS. Files is usually willing to hand a registered document type to its owner app, while Messages often keeps the attachment inside its own interface.

DoodleTag now ships an iOS Share Extension to improve intake from share-sheet hosts:

  • DoodleTag can appear in the iOS share sheet for compatible .dtag file, file URL, URL, or generic data shares
  • the extension writes accepted items into the shared app-group inbox
  • the main app imports the pending item when DoodleTag becomes active
This improves the share-sheet route from Messages-like hosts. It does not guarantee that tapping an attachment bubble in Messages will open DoodleTag directly.

Current Product Position

  • macro sharing remains file-based
  • the primary shared artifact is the raw .dtag file
  • the iOS Share Extension is part of that installed-device file-sharing workflow
  • sharing is still one-to-one between devices that already use DoodleTag

Future Direction: Upload to One Server and Share the Link

The next larger direction would be a separate public-sharing path:

  1. a user records or exports a macro
  2. DoodleTag uploads the .dtag file to one server
  3. the server returns a public URL for that file
  4. DoodleTag shares a web link that points to the uploaded macro

The strongest version of that flow would use a doodletag.500nits.com landing URL rather than exposing the raw storage URL directly.

Recipient Behavior in the Future Model

If DoodleTag Is Installed

  • the shared link can route into DoodleTag through the existing URL handling path
  • DoodleTag can resolve the real .dtag
  • DoodleTag can download the file and show the normal import or play flow

If DoodleTag Is Not Installed

  • the shared link opens in the browser
  • the browser can land on a DoodleTag page
  • that page can explain what the macro file is
  • that page can send the user to the App Store

This is the growth effect of the public-sharing model: the public share can reach existing users and people who do not yet have DoodleTag installed.

The Server Side of That Future Direction

The simplest implementation shape is:

  • one public object-storage bucket
  • one random object path per uploaded macro
  • one public URL returned after upload

Simple storage choices include Cloudflare R2, Amazon S3, and Supabase Storage. A minimal object path could look like macros/2026/03/<random-token>.dtag.

The important detail is not the exact vendor. It is that the path must be long, random, and hard to guess.

Privacy and Product Implications

Public link sharing means anyone with the link can access the macro, recipients can forward it, and the link may appear in previews, logs, analytics, or other third-party systems.

The private one-to-one .dtag format can contain creator or device-related information, timestamps, and detailed stroke or event history.

That is why a public-sharing flow should use a dedicated sanitized export instead of uploading the private exchange file verbatim.

  • use random, non-guessable object names
  • make the UI explicit that the generated link is public-by-link
  • strip creator identity, device-related fields, and any other sensitive or protected metadata before upload
  • consider later support for expiration or deletion

Recommended Future Shape

The cleanest product split is to keep the two sharing modes separate.

Share File

  • direct device-to-device exchange
  • best for installed users
  • uses the raw .dtag file

Share Publicly

  • upload to one server
  • share a public URL
  • best for Messages, social sharing, and recipients who may not have DoodleTag installed
