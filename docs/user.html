1. Canvas Fundamentals

  • Infinite-session style drawing on a fixed paper world with pan and zoom.
  • Single-tap viewport reset returns to full paper view.
  • Long-press viewport reset cycles zoom presets (view one, view two) for quick framing.
  • Drawing input outside the paper is ignored.
  • Undo and redo are personal action stacks. Tap Undo to remove your latest undoable drawing/tool action; long-press Undo to redo your latest undone action.
  • In Glue, each doodler keeps their own undo/redo flow. When one doodler undoes or redoes an action, the peer receives the matching canvas update for that doodler’s action.
  • The Eraser is undoable because it stores the exact line cuts it makes: the original affected line geometry, the surviving replacement fragments, and the stacking order needed to restore the scene. It only affects finalized lines.
  • In solo mode, Air Blower actions are undoable as snapshot-backed _UndoType.airBlower actions. In Glue, Air Blower commits the changed shared canvas as a fresh baseline and clears undo/redo history for the shared session.
  • Recording playback is a solo-only controlled operation. Completed playback adds one snapshot undo action, so undo restores the pre-playback canvas and redo restores the fully played playback result. Canceling playback restores the canvas that was visible when playback started.

2. Main Controls (Tap vs Long-Press)

Most controls in DoodleTag have two gestures. A normal tap is the quick everyday action. A long press means touching and holding the control for a moment until its secondary action opens. In practice, most buttons are designed with one direct tap behavior and one longer-press behavior, so it is worth trying both when learning the app.

3. Glue (Collaborative Canvas)

Glue is DoodleTag’s shared drawing mode. It connects two devices into one live session so both people work on the same collaborative scene. The pair canvas means the shared canvas that belongs to that exact pair of devices. It is separate from each device’s solo canvas, so entering Glue does not overwrite your private drawing.

3.1 Glue Rules

  • Glue is one-to-one: one active peer at a time.
  • Each device pair has its own persistent shared canvas.
  • Entering Glue saves/parks the current solo canvas locally and switches to the pair canvas for the connected device pair.
  • Glue always shows the pair canvas for that specific pair.
  • Leaving Glue saves the pair canvas and restores the solo canvas.
  • If the pair has no saved shared canvas yet, Glue starts from a blank shared canvas for that pair.
  • If the same two devices Glue again later, DoodleTag restores their previous shared pair canvas.
  • Undo/redo history does not travel between solo and Glue canvases. After DoodleTag restores a solo or pair canvas, undo/redo starts clean for that restored canvas.

3.2 The Glue Process Layer

When you tap Peers, DoodleTag opens a Glue process layer. This layer may appear only briefly if Glue connects quickly.

The layer shows simple, readable progress messages such as:

Searching for doodlers...
Found Tycho Nokia G42 5G
Dialing Tycho Nokia G42 5G...
Receiving a doodler...
Connected with Tycho Nokia G42 5G
Restoring your shared drawing...
Shared drawing restored.

While this layer is visible, DoodleTag blocks other actions that could interfere with the Glue operation.

3.3 Canceling Glue

The Glue process layer has its own Cancel button.

Cancel does not dismiss the layer instantly. After pressing Cancel, DoodleTag must finish the cleanup first. During cleanup, the layer may show messages such as:

Canceling glue...
Saving shared drawing...
Returning to your canvas...

Only after cleanup has finished does the layer disappear and unlock the rest of the app.

3.4 Glue Needs Attention

If the layer says Glue needs attention, the Glue process hit a real connection or cleanup problem and needs the user to let it finish or press Cancel.

This state is meant for explicit problems such as:

  • the connection could not be made
  • the network/socket failed
  • Glue startup / cleanup failed

3.2 Glue Behavior and Restore

  • Device discovery/connection for shared drawing sessions.
  • Auto-glue session restoration and reconnect behavior with peer/session memory where applicable.
  • Each device pair has its own shared canvas snapshot, and the same pair restores that shared canvas on reconnect.
  • Entering Glue preserves the local solo canvas in a separate restore slot instead of merging solo and shared history.
  • Leaving Glue saves the pair snapshot and restores the solo snapshot.
  • Snapshot metadata/data exchange keeps peers converged on the newest shared state and avoids stale overwrites.
  • Long-pressing Peers forgets the last preferred peer without deleting snapshots.

4. Drawing and Color Features

  • DoodleTag stores drawing as vector lines and segments. Each mark keeps its own geometry, point path, line size, cap, color, and stacking order instead of being flattened into one bitmap.
  • In practice, this means every piece of the drawing remains a structured part of the scene. Zoom stays clean, Glue can synchronize authored lines, playback can replay recorded drawing actions, undo can restore individual operations, and tools like the Eraser or Air Blower can affect line geometry without reducing the whole canvas to one merged image.
  • Compared with simpler drawing apps where marks are treated more like immediate screen output, DoodleTag keeps the drawing as a live composition of line objects. That is why the same authored drawing can be restored, replayed, shared, cut, moved, recolored, or re-layered through later tools and sessions.
  • Standard freehand drawing with pencil size presets and a precise size slider.
  • Pencil/tool size is not one global number that every tool shares. Each supported tool/context carries its own remembered size, and switching away does not overwrite the sizes you previously chose elsewhere.
  • The practical rule is: when you return to a tool/context, DoodleTag restores the last size that belonged to that same tool/context.
  • This preserves tool feel. A tool that makes sense as delicate can stay delicate, while a tool that makes sense as broad can stay broad, without forcing you to re-tune size every time you switch.
  • The remembered size contexts currently documented in the product are:
    • normal drawing
    • drawing with orthogonal square-grid snapping
    • drawing with orthogonal hex-grid snapping
    • eraser
    • rainbow
    • rainbow with orthogonal square-grid snapping
    • rainbow with orthogonal hex-grid snapping
  • Example: set normal drawing to extra thin, switch to the Eraser and make it much wider, then switch back to normal drawing. Normal drawing comes back at its own thin width; it does not inherit the Eraser’s width.
  • Example: set the Eraser to a narrow erase width, switch to normal drawing and make the Pencil large, then switch back to the Eraser. The Eraser comes back with its own remembered erase width.
  • Example: set Rainbow to a bold width for expressive color lines, then switch to normal drawing and keep that Pencil narrow for sketch details. Returning to Rainbow brings back the bold Rainbow width instead of the narrow sketch width.
  • Example: set orthogonal square-grid drawing to a thin guide-line width, then switch to freehand drawing or the Eraser and use thicker marks there. Returning to orthogonal square-grid drawing restores the thin guide-line width that belonged to that snapped context.
  • Example: set orthogonal hex-grid drawing to one width and orthogonal square-grid drawing to another. Those snapped contexts can keep different remembered widths, so moving between them does not force one grid mode to borrow the other’s size.
  • Example: rainbow freehand, rainbow orthogonal square-grid, and rainbow orthogonal hex-grid can each keep different remembered widths. Returning to one of those rainbow contexts restores that context’s own last width rather than reusing the last rainbow-related width from somewhere else.
  • In short: each supported tool/context carries its own size memory, so switching tools feels like resuming that exact mode rather than reconfiguring a single shared Pencil setting.
  • Quick palette includes:
    • Palette tab for fast swatch picks.
    • Wheel tab for continuous color selection.
    • Recent color history integration.
  • Pencil setup opens from long-pressing Line Size.
  • Pencil setup controls the current pencil/tool size with a slider and the same preset values used by the quick tap.
  • The Plain and Outline choices inside Pencil setup are tool selectors:
    • Plain selects normal drawing.
    • Outline selects Outline Pencil.
    • Selecting Outline deselects Rainbow, Eraser, and Air Blower for drawing.
    • If Pencil setup opens while another tool is active and Outline is selected temporarily, selecting Plain in that same open form returns to the tool that was active when the form opened.
    • Outline is selected only while Outline Pencil is the active drawing tool.
    • Tool Shouts announce selector choices, such as OUTLINE PENCIL!, DRAW, or the restored tool name.
  • In Outline Pencil, the selected size is the final outside size of the mark. Border size is taken out of the inside fill, so Plain and Outline at the same size keep the same overall footprint.
  • The quick palette stays focused on color editing:
    • In Plain, the palette and wheel edit the main pencil color.
    • In Outline Pencil, the inside color comes from the current plain pencil color.
    • In Outline Pencil, tap the bordered preview circle to switch which color you are editing: one tap selects the inside color, the next tap selects the outer border, and so on.
    • In Outline Pencil, the palette, wheel, and history edit whichever part of that preview is currently selected.
    • When you change the inside color, DoodleTag automatically uses the default outline tuning: a 40% darker border with a 17% border size.
  • Rainbow color editing is opened from the Rainbow tool path, or by tapping Color/Palette while Rainbow is active. The Rainbow color editor keeps Rainbow selected.
  • Closing the quick palette does not switch tools.
  • Recent-color behavior is stable (reselecting colors does not unexpectedly delete history entries).
  • Blue/green pair colors are used only for first-time Glue defaults; user-selected colors remain user-owned.

5. Tool Interactions and Exclusivity

  • Square Grid and Hex Grid are mutually exclusive.
  • Eraser, Air Blower, and Orthogonal are mutually exclusive in direct tool toggles.
  • Enabling Rainbow turns off incompatible tools where needed and keeps rainbow-specific line behavior active.
  • Orthogonal snapping behavior depends on guides:
    • If square grid is active: snaps to grid-aligned orthogonal directions.
    • If hex grid is active: snaps along hex-edge directions.
    • If no grid is active: snaps to orthogonal directions in free space.

6. Rainbow Tool

  • Toggle rainbow on/off directly from the menu.
  • Normal rainbow follows a toothpaste-style model: colors are stored in a fixed internal sequence and come out from the point where the line starts. Reversing the draw direction remaps those colors around the shape instead of preserving the same edge-to-color assignment.
  • Rainbow color progression is distance-based, not time-based. Moving faster or slower over the same path does not change the internal color order; it only changes how quickly that ordered color stream is laid down along the path.
  • Styles:
    • Normal
    • Pop
    • Lanes
  • Segment length is adjustable with step controls for Normal and Pop; it is disabled for Lanes.
  • Palette control:
    • Six editable rainbow palette slots.
    • Reset to default rainbow palette.
    • Uses the dedicated rainbow color editor.
    • Long-press Rainbow, or tap Color/Palette while Rainbow is active, to tune Rainbow colors without leaving Rainbow.
  • Lanes style supports lane-aware turning/join behavior and ongoing geometry refinements.
  • Rainbow settings persist across app restarts.

7. Eraser

  • Toggle the eraser and erase by dragging/tapping across lines.
  • The Eraser is geometry-based: it does not paint transparent pixels over the canvas. Instead, it traces your erase path through the stored line geometry and cuts intersected line segments out of the scene.
  • In practice, each erase move is treated as a short line segment, and the full drag becomes a chain of segment cuts. A tap works like a tiny centered cut at that point.
  • Eraser settings are configurable via the long-press settings dialog.
  • Eraser size controls the visible eraser band, the narrow hit area used to decide which line geometry is touched, and the maximum gap the eraser can cut.
  • Eraser strength controls how much of that selected size is applied after a hit.
  • Higher eraser strength uses the full selected eraser size, so a small eraser remains a small precision cut even at high strength.
  • Lower eraser strength cuts a smaller gap inside the selected size for finer trimming.
  • On long lines or large shapes, the eraser usually splits the original line into smaller surviving pieces on either side of the removed gap.
  • On short segments or tiny fragments, the removed gap can consume the whole remaining piece, so that fragment disappears entirely instead of being split into even smaller parts.
  • Because the Eraser works on stored line geometry, the surviving pieces remain normal vector lines: they still participate in zoom, replay, Glue sync, and later editing instead of becoming flattened bitmap damage.
  • The Eraser preserves stacking by replacing affected lines with the resulting cut pieces in the same scene position, so later overlapping content continues to layer consistently.
  • Multi-lane rainbow lines are erased as one coordinated lane bundle, so sibling lanes break together instead of each lane being cut independently.
  • Eraser undo/redo is based on the exact line transformation: the app records the original affected line geometry, the replacement fragments that survive the cut, and the archive-order data needed to restore stacking on undo and redo.
  • Undoing an erase removes the replacement fragments and restores the original affected lines. Redoing the erase removes the restored originals and brings back the cut fragments.
  • The Eraser works only on finalized stored lines. In-progress active lines are allowed to finish before they can participate in later geometry erasing.
  • Long erases show a progress overlay.
  • Canceling a progress erase restores the canvas state from the start of that erase gesture.

8. Air Blower

  • Toggle blower mode for playful line displacement, removal, and color-mixing effects.
  • Long-press settings include:
    • Radius
    • Strength profile
    • Kind:
      • Displace
      • Vortex Remove
      • Mix Colors
  • In solo mode, Air Blower actions are undoable as snapshot-backed _UndoType.airBlower actions.
  • In Glue or remote application, Air Blower commits the changed shared canvas as the current undo/redo baseline for that shared session.
  • In Glue, an Air Blower baseline commit clears the shared undo/redo history before later shared actions begin a fresh chain.
  • Tap and drag blower gestures both follow the same split rule: solo gestures remain undoable, while Glue drag gestures commit one shared-session baseline when the gesture completes.
  • Hotspot zap erasures count as Air Blower canvas changes, so they participate in the same solo-undo vs Glue-baseline rule.
  • Remote/glued playback applies blower actions consistently on both devices.

9. Recording System

9.1 Recording and Sharing

  • While solo, tapping the recording control while idle first opens Record your drawing.
  • While glued and connected, recording starts only when both peers hold REC together; the solo recording form does not appear in that shared flow.
  • Starting a recording snapshots the current canvas, clears it to blank for recording, and restores the previous canvas when the recording ends.
  • Once recording starts, your drawing is captured on a blank canvas.
  • Recording captures:
    • Finger-driven drawing/tool events
    • Tool setup at the moment of use
    • Timeline events
    • Viewport metadata
  • Stopping a recording saves that finished .dtag recording file in the Doodle Tag folder, restores the canvas that was visible before recording began, and reports that the recording was saved.
  • Once a recording has been saved, the Export sheet’s Video section offers Share last recording, which renders the last saved .dtag into an .mp4 on the device before opening the system share sheet.
  • Long-pressing the Export button skips the sheet and shares the last saved .dtag recording file directly through the system share sheet. With no recording saved, this shouts NO RECORDING SAVED.
  • Video export shows a progress form with a Cancel button. Canceling stops the export and removes any temporary frame/video files created for that attempt.
  • Recorded .dtag files are saved with their canonical file name directly in the Doodle Tag folder.
  • While recording, your drawing stays in step with what you are shaping on screen: keep what you like, roll back what you do not, and continue refining before you stop.
  • Playback focuses on the finished drawing you crafted, without turning playback into a visible sequence of backstage corrections.
  • Clear Canvas is disabled while recording so the drawing remains a pure event stream without canvas reset events.
  • Export/share flow produces .dtag recording files.

9.2 Recording File Format and Compatibility

  • .dtag uses password-protected ZIP packaging (Doodle Tag) with compatibility handling for legacy payloads.
  • Playback supports bundled assets, local files, and deep-link driven downloads.

9.3 Playback Behavior

  • Recording playback is globally locked to one active playback at a time.
  • Playback runs through a read-only long-task UI.
  • Playback starts from a blank canvas and resets the view so the saved drawing replays on its own.
  • User controls during playback:
    • Skip fast-forwards to the end of the recording.
    • Cancel restores the canvas that was visible when playback started.
  • Skip/cancel reset viewport framing consistently.
  • When playback completes, the fully played drawing is added as one undoable snapshot playback result. Undo restores the pre-playback canvas, and redo restores the fully played recording result.

10. Welcome Tour

  • Dedicated ? utility button launches the bundled welcome tour recording.

11. Import, Deep Links, and File Handling

  • OS-level .dtag opening support on Android and iOS.
  • DoodleTag recordings are saved as .dtag files.
  • On iPhone and iPad, .dtag files can be opened with DoodleTag through supported file flows such as the Files app.
  • Not all apps handle .dtag attachments in the same way. Some apps, such as Files, can pass the file to DoodleTag directly. Others, such as some messaging apps, may keep the file inside their own preview or sharing interface.
  • If a .dtag file does not open directly from a messaging app, try saving it to Files or sharing it from there into DoodleTag.
  • On iPhone and iPad, that recording folder appears in Files as On My iPhone > Doodle Tag.
  • On Android, recordings are saved in Download/Doodle Tag.

12. Export and Long Tasks

  • Tap Export to open the Export bottom sheet. The sheet has:
    • An Include grid toggle (off by default) that controls whether the active grid is rendered into the exported PNG.
    • Three size presets that immediately start the image export when tapped: Small (1024px), Normal (2048px), Max (4096px).
    • A Video section that, when a saved recording exists, offers Share last recording to render that recording into an .mp4. When no recording exists, the section shows No recording saved yet.
  • Image export saves the PNG to the device Photos library under the Doodle Tag album and also writes a copy into the app’s exports/ folder. The export adds a small Doodle Tag watermark to the image.
  • Dismiss the sheet by tapping outside it; size-preset and Share last recording taps start their action immediately.
  • Export uses controlled progress flows to avoid partial/interrupted output.
  • Long-running operations use shared long-task overlay patterns for consistent UX.
  • Long-task surfaces are integrated with the bottom menu replacement when applicable to reduce canvas obstruction.

12.1 Video Export Expectations

  • Video export is intentionally light and playful rather than a production-grade rendering system.
  • The MP4 is produced on the device itself, so results depend on the platform and the specific phone or tablet doing the work.
  • Android and iPhone do not use the same native encoder stack, so the result will be different.
  • Encoding can fail on some devices or for some recordings even when playback itself works correctly.
  • Video smoothness is influenced by device CPU performance, so older or weaker devices may produce less fluid results.
  • Use recording video export as a for fun only sharing feature.

13. Persistence and Restore

  • Persisted settings include:
    • Color, size, cap
    • Per-tool/per-context size memory so each drawing mode can return to its own preferred feel
    • Recent colors
    • Grid/hex parameters
    • Rainbow settings
    • eraser/blower/tool toggles and related parameters
    • Shout banner preference
    • Glue preferences, session linkage, and per-peer shared canvas restore data

14. Reset and Recovery Actions

  • DoodleTag reset actions are split by domain so you can clear only the part you mean to reset.

14.1 Reset All Tool Settings

  • Long-press the Tool Shouts megaphone button.
  • This opens Reset all tool settings?.
  • It restores default values for tool-facing state such as:
    • Pencil/tool size values
    • palette-related tool settings
    • rainbow settings
    • eraser / air blower settings
    • grid / hex-grid settings
    • shout-banner mode
  • It is the closest thing to “reset my tools back to defaults”.
  • It does not clear saved recording files, onboarding flags, or saved session/snapshot files.

14.2 Clear Saved Sessions and Snapshot Restore Data

  • While not glued, long-press Clear Canvas.
  • This opens the saved-session clearing confirmation flow.
  • It deletes the persisted snapshot/session storage used for:
    • latest solo canvas restore
    • parked solo restore snapshot
    • saved per-peer Glue pair canvases
    • stored Glue session restore metadata/preferences
  • It keeps the current live canvas intact and only removes the stored restore/session data.
  • After the wipe, DoodleTag shouts GLUE SESSIONS CLEARED.
  • Use this when you intentionally want to remove remembered canvas/session state, including saved shared pair canvases, not just erase what is currently visible.

14.4 Reset Welcome Tour Onboarding

  • Long-press the Welcome Tour (?) button.
  • This clears the “tour already played” flag.
  • Use it when you want the welcome tour to be eligible again like a first-run onboarding prompt.
  • This does not change drawing tools, snapshots, or recording storage.

14.5 Undo/Redo Recovery Boundaries

  • Undo and redo are meant for recent personal drawing and geometry edits.
  • Eraser actions are undoable because they store the exact line cuts they made, including the replacement fragments and the stacking order needed to restore the scene.
  • Completed recording playback stays undoable as one snapshot operation, with undo restoring the pre-playback canvas and redo restoring the fully played result.
  • Restoring a solo canvas or Glue pair canvas also starts with clean undo/redo history for that restored canvas.

15. Current UX Notes

  • Main menu and utility controls expose accessibility labels, and the compact top-right utility buttons keep a larger effective tap target than their visible size.
  • Tool shout banners use a 3-level megaphone control: all banners, announce-only, and silent.
  • Selecting a tool now emits an announcing banner using the same shout surface as live tool usage.
  • Toggle-style tools announce explicit state labels such as GRID ON, GRID OFF, HEX GRID ON, RAINBOW ON, and ORTHO SNAP OFF.
  • Rainbow announcements are style-aware and distinguish RAINBOW, POP RAINBOW, and LANES RAINBOW.
  • Tool shout banners are uppercase-normalized at render time.
  • Welcome tour and playback surfaces share behavior and controls.
  • Some internal diagnostics overlays exist for development/testing and are not part of regular drawing workflows.
